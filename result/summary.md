# 국가기초학력지원포털 프론트엔드 작업 요약 보고서

본 문서는 현재까지 진행된 사용자 인터페이스(UI) 및 프론트엔드 라우팅 고도화 작업에 대한 요약입니다. 백엔드 연동 전, 프론트엔드 주도 개발(Frontend-First Development) 방식으로 전체적인 와이어프레임과 화면 이동 흐름을 완성했습니다.

## 1. 전역 네비게이션 바 (Global Navbar) 도입
- **위치:** `src/components/Navbar.tsx`
- **구현 내용:**
  - 기존 각 페이지(`Home`, `Entry`, `StudentLogin`)에 흩어져 있던 네비게이션 로직을 통합.
  - 마우스를 올리면(Hover) 자연스럽게 열리는 전체 메뉴(Dropdown) 레이아웃 적용.
  - 상단에 고정(Sticky top)되어 사용자가 페이지를 스크롤하더라도 항상 메뉴 접근이 가능하도록 구성.
  - 접속자의 권한(게스트 vs 로그인된 교사 등)에 따라 우측 회원 정보란이 동적으로 변하도록 상태(Mock) 적용.

## 2. 권한별 진입점(Entry) 통일성 강화
- **위치:** `src/pages/Entry.tsx`
- **구현 내용:**
  - 메인 통합 [로그인/회원가입] 버튼 클릭 시, 회원 유형(학생, 교사, 일반회원)을 선택할 수 있는 Entry 페이지 개선.
  - 학생 클릭 시 학생 전용 로그인 창(`/student-login`)으로 즉시 이동.
  - 교사 클릭 시 교사 전용 업무 대시보드(`/teacher-dashboard`)로 부드럽게 라우팅되도록 연결.

## 3. 교사 업무 전용 대시보드 구축 (Teacher Dashboard)
- **위치:** `src/pages/TeacherDashboard.tsx`
- **구현 내용:**
  - **초기 홈 화면 (Dashboard Home):** 접속 직후 6개의 주요 교사 업무(학교정보, 검사설정, 시험감독, 통계, 학습관리, 자료실) 버튼이 메인 영역에 넓게 펼쳐져 직관적인 접근 제공.
  - **2단 분할 레이아웃 적용:** 화면 좌측에 아코디언(Accordion) 스타일의 사이드바를 배치하여, 어떠한 세부 업무를 진행 중이더라도 다른 업무로 쉽게 이동할 수 있는 전문적인 관리자(Admin) 형태 구축.
  - **권한 기반 맞춤형 UI 제공 (Mock):**
    - **학교정보 > 학생관리:** 학년반별 학생 목록, 가입 여부(가입완료/미가입 확인), 아이디 일부 마스킹(stu***), 초기화 버튼 구현.
    - **시험감독:** 배포가 완료된 진단 검사만 노출되도록 하는 직관적인 안내창과 학년반 검색 필터링 테이블 구성.

## 향후 진행(Next Steps) 과제 제안
1. **백엔드(Spring Boot) 연동 준비:** 작성된 목업(Mock) 데이터와 화면 상태를 실제 REST API(`axios` 사용)와 연동하여 동적인 데이터를 그려냄.
2. **JWT 기반 로그인 인증/인가 로직 구현:** 로그인 엑세스 토큰을 기반으로 사용자 정보와 역할(학생, 관리자, 담임, 교과교사)을 전역 상태로 관리.
3. **추가 세부 화면 기획 적용:** 검사설정 내의 '국기초, 맞자평' 등 세부적인 화면 기획이 결정되면, 만들어진 사이드바 프레임워크 안에 테이블/폼 UI를 추가.
